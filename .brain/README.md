# .brain — Unified Learning System

This folder is the single source of truth for everything learned across all client website builds, automation projects, and pitches. Every session should read from here before starting and write back after completing.

## Structure

```
.brain/
├── README.md              ← You are here
├── mistakes.md            ← Every mistake made, categorised, with fixes
├── wins.md                ← What worked well — validated patterns to repeat
├── design-system.md       ← Colour, font, layout patterns per industry
├── tech-patterns.md       ← Code patterns, bugs, and solutions
├── conversion-rules.md    ← What converts visitors to leads/calls
├── competitor-research.md ← Research findings from competitor analysis
├── client-checklist.md    ← Pre-build checklist before starting any project
├── build-log.md           ← Chronological log of every project built
└── 3d-websites.md         ← Premium 3D/cinematic site playbook (Three.js, Lenis, nano-banana, perf)
```

## Rules

1. **Read before building.** Every new project starts by reading `mistakes.md` and `client-checklist.md`.
2. **Write after building.** Every completed project adds to `wins.md`, `mistakes.md`, and `build-log.md`.
3. **No duplicates.** If a lesson already exists, update it with the new context — don't add a second entry.
4. **Be specific.** "Don't use wrong colours" is useless. "Don't add amber to a monochrome brand — BitumenCo incident" is useful.
5. **Include the why.** Every entry explains why the decision was good/bad, not just what happened.
