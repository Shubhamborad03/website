# Client Checklist — Before Starting Any Build

Complete this before writing a single line of code.

---

## 1. Brand Extraction
- [ ] Download logo (verify file format with `file` command)
- [ ] Create light and dark variants of logo
- [ ] Extract exact colours from existing site/logo/social
- [ ] Identify: is the brand monochrome or does it use accent colours?
- [ ] Screenshot their current site for reference

## 2. Content Verification
- [ ] Check Google reviews — count and rating
- [ ] Check Facebook page — exists? Reviews?
- [ ] Check Instagram — exists? Content quality?
- [ ] Verify ABN on ABR
- [ ] Verify QBCC licence (if trade)
- [ ] Note real testimonials available (zero = no testimonial section)

## 3. Business Understanding
- [ ] Who are their customers? (residential, commercial, both?)
- [ ] What's their #1 service? (lead with this)
- [ ] What's the buyer's #1 question? (answer in first 2 scrolls)
- [ ] What's the buyer's #1 objection? (address early)
- [ ] Service area — exact suburbs/regions
- [ ] Pricing — transparent or quote-based?

## 4. Competitor Check
- [ ] Search "[service] [location]" on Google
- [ ] Note top 3 competitors and what their sites do well
- [ ] Identify gaps Wolf Partners can exploit

## 5. Technical Setup
- [ ] Create project folder with correct name
- [ ] Next.js App Router + Tailwind (check version)
- [ ] Copy images to /public
- [ ] Set up proper metadata (title, description)
- [ ] Verify fonts load correctly

## 6. Design Decisions (before coding)
- [ ] What section order fits THIS industry's buyer journey?
- [ ] Dark or light theme? (match industry/brand)
- [ ] What font pairing? (reference design-system.md)
- [ ] Homepage form placement decided
- [ ] Mobile CTA strategy decided

## 7. Content Rules
- [ ] All images sourced from client (zero stock)
- [ ] All testimonials verified as real (zero fabricated)
- [ ] All stats/credentials verified
- [ ] Service area data from client (not guessed)
- [ ] Contact form has a working backend or mailto fallback
